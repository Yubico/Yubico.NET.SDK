# Live Documentation Governance

This repository has multiple documentation lanes. Each lane has a different source of
truth and a different automation contract. The goal is small evidence-backed docs PRs,
not broad generated rewrites.

## Lanes

| Lane | Scope | Source of truth | Automation | Failure mode |
| --- | --- | --- | --- | --- |
| Docs hygiene | Active docs only | `DiscoverActiveDocumentationFiles()` in `toolchain.cs` | `dotnet toolchain.cs -- docs-qa` | Broken links, unbalanced fences, known stale examples |
| Architecture diagrams | `docs/architecture/**` | `sdk-architecture-map.yml` + Mermaid source blocks | `dotnet toolchain.cs -- docs-architecture` and architecture-docs workflows | Evidence drift, stale rendered images, unmapped diagram boxes |
| Migration docs | `docs/migration/**` | v1 `develop` vs v2 `yubikit` context | migration-docs workflows | v1-to-v2 API guidance drift |
| Inventory | Active docs report | `dotnet toolchain.cs -- docs-list-active` | `dotnet toolchain.cs -- docs-inventory` | Review signals only; never auto-rewrite |

## Active Docs Boundary

`docs-qa` defines the active documentation set. Inventory tooling must consume
`dotnet toolchain.cs -- docs-list-active`; it must not independently reimplement the boundary.

The active set includes root Markdown, top-level `docs/*.md`, `docs/usage/**`,
`docs/troubleshooting/**`, `docs/architecture/**`, and module `README.md` / `CLAUDE.md` files
under `src/**`. It excludes archive/planning/research/spec/template material by design.

## Report-Only Inventory

`docs/docs-inventory-report.md` is a snapshot generated by:

```bash
dotnet toolchain.cs -- docs-inventory
```

The report lists maintainer-review signals such as old last-touch dates, large files,
task markers, and lane ownership. These signals are not failures. They are triage input
for later focused PRs.

## Rules

- Do not mass-rewrite active docs from an inventory report.
- Keep semantic update lanes narrow: architecture automation edits `docs/architecture/**`,
  migration automation edits `docs/migration/**`.
- Generated architecture images must be committed with `images/image-manifest.txt` from the
  same render.
- Watermarks (`docs/architecture/.state.yml`, `docs/migration/.state.yml`) are lane-owned state,
  not general documentation content.
- A docs PR is complete only when `docs-qa` passes. Architecture changes must also pass
  `docs-architecture`.
